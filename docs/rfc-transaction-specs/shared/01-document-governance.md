# Shared Requirement: Document Governance

## Purpose

Define the minimum governance standard for all transaction RFCs.

## Required Metadata

Every transaction RFC must include:

- document id
- title
- version
- status
- owner
- reviewers
- approvers
- last updated date
- effective date
- related documents

## Change Log

Every transaction RFC must maintain a change log with:

- version
- date
- author
- summary of changes

## Status Values

Allowed status values:

- Draft
- In Review
- Approved
- Implemented
- Deprecated

## Rule

No transaction RFC is considered normative unless it has a document id, version, and status.
